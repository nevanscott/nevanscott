A Report from the Future ∞

Well I’m sitting here at LAX at a Samsung Mobile Charging Station, so I thought it was as good a time as any to start reflecting on the decade up until this point.1 So far, I’ve used my Visa to cover magazines and Sudafed from a Hudson News, which didn’t carry Coke, so I bought a soft drink with my Visa from Burger King where using a card was advertised as being “faster than cash” and I had to work a bit to figure out how to get both ice and Coke out of the machine that we are told is a fountain but bears little resemblance to my eyes.

Brand and commercial pollution are nothing new, but I hold out hope that these things are subsiding. The whole idea of large corporate entities just seems flawed in this world. It has served us well in the way of making goods so convenient that people are open to focusing time and energy on more specific things. TK But in many ways, commercialism and new technology have paved the way for a less-informed society that is treated to an incessant barrage of advertisements and other entreatises to spend money. That’s what our current system more or less depends on. This is one of the reasons it is harder for me to venture out in the world. Malls and movie theatres and even airports are so full of things that, simply put, overwhelm and confound, that it’s just not worth going on a regular basis.

Moving Web Type Forward ∞

In recent years, a lot has been done to improve the state of typography on the web, and it shows. It is actually possible to read online on a regular basis, or print newspapers wouldn’t be struggling as much as they are to transition into a new economic system.

Luckily, for those of us who are energized by the renaissance of the visual word, there is still much to be done. To add to the excitement, many of the things needed to move things forward are probably going to take joint effort from a variety of specialties. So, let’s get started.

Hyphenation

When type was set in medal, the process of hyphenation and justification was literally in the hands of the typesetter. On the web, the typesetter is the browser, which renders type guided by CSS rules. The browser knows where text is broken onto new lines when the producer of the site does not, partially because the browser knows about the device that it is rendering text on. Fonts may be substituted, sizes may be altered, the width of the view may be narrower or wider. This is as it should be: the browser should handle typesetting.

At the moment, the following CSS rule will ask the typesetter to justify paragraph text:


p { text-align: justify; }

This isn’t a trivial problem to solve,

This is some information about the kind of stuff we need going forward.

As a quick aside before getting into this list, I should point out that I think for the most part the issue of web fonts is a bit of a distraction from a lot of other problems that are less sexy because they’re (intentionally) less visible to most users.

Here are the real things that would be helpful immediately:

We need a good dictionary system to work in conjunction with web browsers to feed them hyphenation information for different languages and implementation across the board of a standard CSS rule for giving the browser instructions for how to hyphenate. This also needs to be really fast.

We need simple CSS rules for giving instructions about widows and their handling. We also need to be better able to give an indication regarding orphaned lines in print media. Both of these should be handled in a reasonable way by the defaults for the browser.

Automatic detection of ligatures and CSS rules for OTF features such as lining, text, and tabular figures; stylistic alternates; discretionary ligatures; true small caps; etc.

There should be a way to specify whether text is vertically centered within the line-height as it is now, or instead attached to the baseline as expected by most print designers. For now, the default should be left. This is less critical than the above.

Along with the things mentioned, we also really need at least one solid typeface to work with. That’s what is so encouraging about the work currently being done to refresh the ubiquitous Georgia typeface. I can only hope that the family will be free to download and easy to install, or possibly that it might be bundled with the browsers in some kind of update or something. Without a free typeface installed on a vast majority of machines, in order to take advantage of open-type features we would need a consistent and stable way to embed such a font (even for a fee) into websites.

Either way we need at least one rock-solid typeface with support for internationalization and al of the open-type features in order to avoid a separation between websites that can afford a budget for fonts and those that cannot.

Abstracting Away the File System ∞

When I was in school, my computer science professor was fond of saying, “Bits is bits.” What he meant is that all of computer science is an abstraction.1 When thinking about computers, it is important to keep the principle of abstraction in mind. Without going into too much detail, computers are essentially made up of ones and zeros. The idea of the filesystem, the one that most users interact with through a graphical operating system, is a metaphor or abstraction that the operating system provides to the user and to the applications that the user interacts with.

As we approach another decade in computing, it might be worth asking why our computers organize information in this way and why it is presented in such a fashion to the user. I know people, myself included, who still use pieces of paper that can be filed away in a real folder or sometimes stored more permanently in a file cabinet. But modern use of those organizational tools is significantly lower than it was two decades ago. Some people my age are entering the workforce having spent more time interacting with their computer’s file system than they have spent looking at sheets of paper, much less using them or storing them away.

So why continue to think about the data storage using the same filesystem metaphor, especially in cases where another metaphor may make more sense? TK I believe this is one of the reasons for the early success of iTunes. When Napster popularized music downloading and made MP3 a household word, one of the biggest problems was trying to make these files fit into a storage metaphor based on organizing business documents. Everyone was coming up with their own systems for keeping these files organized. Some would create folders for every artist and every album; others might split up files by genre first, then by artist; some (like me) tended to keep everything in a flat structure and rely on filename conventions like Cream_White_Room.mp3 to find a track. This was a mess, and clearly a broken system because you could only organize tracks in one way. If you wanted to keep your music organized by album, you couldn’t also organize tracks into playlists unless you made copies of the songs.2 And then what if you wanted to have a single track in more then one playlist? In 1999, I thought The Beatles went with everything—what if I wanted Nowhere Man in playlists for “Happy”, “Sad”, and “Fun”?

I don’t recall if iTunes had figured out how to address these problems when it was first released, but not long after iTunes had figure out a good way to keep the user from every worrying about where their music files were or how they were organized. The solution on the Mac runs somewhat deep: When Apple was putting together OS X, it clearly already had at least an inkling of its “digital hub strategy” in mind, and decided that User folders would include a few folders for media right out of the box, including Music. On OS X, iTunes took advantage of this folder to give users the option to let iTunes keep their music files organized for them.

Ross Carter:

“For documents that are recognized by multiple apps, the user has the power to bind a particular document to any app they wish, using the file inspector in Finder. So at least for this purpose the creator code is not needed.”

Two words come to mind: puh leez. In what universe do people create a file and immediately, whistling with merriment, go to Finder and change the application binding for that document?

As much as this argument makes complete sense in a lot of ways, I find myself having the opposite point of view. There is, to my mind, a good potential reason for this kind of change, and I would welcome it if I were yet able to upgrade to Snow Leopard.

This change accompanies another ongoing change, which is the breakdown in the importance of the filesystem itself, which carries important ramifications across the spectrum of computer users.

I have long not understood the refrain that the Mac is not designed for everyone, as if this is somehow the reason that the Mac only has a small percentage of the market share. It may not be priced for everyone, but that is hardly the same as not being designed for everyone. A contemporary Windows PC, by comparison, has price points in everyone’s range, but is, in my skewed view, not really designed for anyone, except maybe CTOs and gamers.3

The contemporary Mac, by comparison, is designed for a wide range of uses; in my opinion, it is a more general platform than Windows. This is easy to forget, because it is so easy to use a Mac in a very specific manner, which is the beauty of its design. This is because ease of use is, as it should be, useful for everyone, no matter what they are doing. Some people use primarily the applications that come with the Mac: especially iTunes and Safari. These are great tools for people who spend most of their time on their computer either browsing the web or consuming music, movies, and TV shows.

But the Mac has also emerged as a great development platform, not least because of its Unix core. The Mac has a wealth of great text editors available to it, such as BBEdit, TextWrangler, and TextMate, but you can also run editors like vim and emacs. Transmit and Cyberduck are both great and intuitive FTP clients. Expandrive is a great way to use the Finder to interact with remote servers over various protocols. Coda and Espresso are promising web development environments. I can easily run a LAMP, Django, or Ruby on Rails application natively on my Mac for development, and easily translate that work onto a Linux server.4

The Mac is undoubtedly a great platform for anyone working with images, video, and sound. In addition to the stable of Adobe software geared toward these users, there are Apple’s own entries in these fields, some of which are basically industry standards like Final Cut.

Outline

  • The Finder is now only one way to navigate the filesystem.
    • Popular tools like TextMate and Coda give a filesystem view as well, in some cases of a remote filesystem on a server.
    • TextMate, Coda, and Cyberduck have some of their own application bindings, not necessarily the same as those used by the Finder.
  • The filesystem is now only one metaphor among many for organizing bits on a personal computer.
    • Probably the most ubiquitous (and earliest) examples of this is iTunes, which successfully organizes songs into playlists rather than into folders.

John Kricfalusi’s Review of Meatballs ∞

For context, John Kricfalusi created The Ren & Stimpy Show. Here’s his take on the new movie based on Cloudy with a Chance of Meatballs:

I had a tough time sitting in my seat through Meatballs, because what was happening and who it was happening to was not remotely interesting. It’s hard to pace a story around characters with no personality.

But as a cartoonist and designer, there was enough visual interest and unique action throughout the movie that intellectually I found things to stimulate me.

It was an optimistic portent of what could be. It’s basically an undirected film – but one that allowed many of the artists to take nothing scenes and add some kind of cleverness, design and action to the formulaic events being told by the story.

This in itself is so far ahead of an overdirected film (overdirected by executives typically, not by directors that actually have a point of view or style) that stops creativity from happening every step of the way, just so that more stock plot points, filler and bad puns can happen.

I think this kind of thing is enjoyable to read not just because of how harsh it is, but because it touches on something that is really true about the current state of, in my opinion, not just animated films, but most major films. I often find myself settling for the little things in a movie that make it good, rather than expecting something more, maybe in order to seem less like a cynic and an asshole.

But the truth is, there’s a lot of terrible stuff out there, and the industry that has grown around making movies has moved it from an art form into a calculating box office science. I can only remain optimistic by holding onto my belief that the creative juices going into films have stagnated due to lack of competition. Lower production and distribution costs, one hopes, will eat away at the joint monopoly whose long project has been the reduction of creative work to “content.”

Time will tell. Quality endures, but only if it can first find life.

The Guts of a New Machine ∞

Came across this again while digging through the DF archives. Rob Walker profiles the then-two-year-old iPod for the Times:

A handful of familiar cliches have made the rounds to explain this — it’s about ease of use, it’s about Apple’s great sense of design. But what does that really mean? “Most people make the mistake of thinking design is what it looks like,” says Steve Jobs, Apple’s C.E.O. “People think it’s this veneer — that the designers are handed this box and told, ‘Make it look good!’ That’s not what we think design is. It’s not just what it looks like and feels like. Design is how it works.”

(Emphasis mine.)

This idea has, I think, taken root since then, and a respect for the relationship between usability, readability, and visual character on the web in particular has really grown. And yet, some companies seem to continue to try to “copy Apple” in all the wrong ways, drawing the wrong inferences, and learning the wrong lessons.

Think about what you’re making, respect the user and the reader, and make informed decisions along the way to the best of your ability. Setting everything minimally against a white background is not the great lesson of Apple’s success. The takeaway is to place an emphasis on the user in the design process and not to underestimate the value of quality.

Kernest ∞

Kernest is one of the first good options for using embedded fonts in a website. Unlike projects like Cufón or sIFR, Kernest takes advantage of @font-face embedding, which is now supported by Firefox, Internet Explorer, and Safari.

One of the problems with @font-face at the moment is that Safari and Firefox support font formats like OpenType and TrueType, while Internet Explorer only supports Embedded OpenType. The usual workaround is to supply a different format depending on the requesting browser. This is exactly what Kernest takes care of for you.

Not only will it serve the right file type depending on the browser, it will only serve the font files when the request comes from a domain name tied to that particular font. Many of the current selection of fonts are free, but in order to make them available to your domain, you must activate them. Based on my brief use, this is a very simple process, requiring only a user account, and adding new domains is completely effortless.

After activating a font for your domain, you simply link to a stylesheet specific to your domain on their server. It is worth noting that there is no javascript involved, and this is as it should be.

I’ve been trying Kernest out for a few days now, and have found the experience very pleasant all around. I don’t know much about what TypeKit’s model will be like, whether they will have some fonts available free of charge, or if there will still be a charge for the service. If not, Kernest looks like it could be a great free alternative for projects without a budget for fonts.

And in the meantime, I highly recommend giving it a try. There are already several serviceable fonts on the site. I’m currently using Droid Serif from Ascender Corp.

Update: I’ve since switched to Heuristica.
