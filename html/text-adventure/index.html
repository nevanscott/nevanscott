Text Adventure

In anticipation of building an engine for a web-based text adventure game, we’ll start out by discussing HTML and CSS to get a handle on the technical side of building interfaces for web browsers. Then we’ll look at some of the concepts of user interaction more generally, motivating the role of human user-oriented thinking in developing a piece of software, and open discussion into how to apply some of this thinking to the project at hand.

For Wednesday, Nov. 3

In preparation, take some time to work through the following tutorials:

  • HTML Tutorial, W3Schools
  • CSS Tutorial, W3Schools

Breeze through the parts that you’re already familiar with, and don’t spend too much time on sticking points. The point is just to have some familiarity so things don’t look completely foreign when we start talking.

I’ve set up a little example here. Please feel free to use that code as a starting point.

Exercise: Write a single HTML page for an imaginary text-based adventure game, using some CSS to change how the page looks. The page should be an example of what a player might see at any given moment of gameplay – probably a description of where they are and some indication of what their options are for continuing. This is just for practice, so have some fun with it if you have time. Please post online somewhere by Tuesday night, Nov. 2, and email Mike a link to your page. He’ll make sure I can get to them.

For Wednesday, Nov. 10

Read through the following pieces. They’re short, but we won’t have time to discuss everything, so please note a passage or two that strike you, and pay particular attention to any common threads between some of these different takes.

  • Bruce Tognazzini, First Principles of Interaction Design
  • Jakob Nielsen, Ten Usability Heuristics
  • Luke Wroblewski, Visible Narratives: Understanding Visual Organization
  • Julia Hayden, Language: The Ultimate User Interface
  • Curt Cloninger, A Case for Web Storytelling
  • Daniel Burka and David Gillis, Friction Can Be a Good Thing

Exercise: Write another single HTML page with a sample form for creating a new node in the story, using CSS to change how the page looks. Don’t worry about having the values of the form post anywhere or to a database, this is purely an exercise in HTML and CSS.
